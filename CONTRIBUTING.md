# Contributing to TECHKNOWMAD LABS

We welcome contributions to our open-source projects. This guide applies to all repositories under the TECHKNOWMAD-LABS organization.

## Getting Started

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Run any existing tests
5. Commit with a clear message (`git commit -m "Add: description of change"`)
6. Push to your fork (`git push origin feature/your-feature`)
7. Open a Pull Request

## Commit Convention

- `Add:` New features or files
- `Fix:` Bug fixes
- `Update:` Changes to existing functionality
- `Remove:` Deletions
- `Docs:` Documentation only
- `Refactor:` Code restructuring without behavior change

## Code Standards

- Python: Follow PEP 8, include type hints, add docstrings
- All code must pass existing linters and tests before PR
- Include tests for new functionality where applicable

## Reporting Issues

Use GitHub Issues on the relevant repository. Include:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Environment details (OS, Python version, etc.)
